# Security Enablement Playbook

## GHAS readiness
- Confirm licensing and repository eligibility

## Code scanning
- Enable CodeQL or third-party scanners

## CodeQL
- Select languages and run schedule

## Secret scanning
- Enable scanning and push protection

## Push protection
- Block secrets at commit time

## Dependency review
- Require dependency review in PRs

## Dependabot
- Enable alerts and version updates

## Security campaigns
- Plan targeted remediation campaigns

## Alert triage workflow
- Define ownership and SLAs

## Security SLAs
- Define severity response times

## Developer education
- Provide secure coding labs and guidance

## Exception management
- Require security review for exemptions

## Metrics
- Alerts opened/closed, SLA adherence, recurrence
