# Lab 02: Repo Inventory

## Objective
Create a repository inventory.

## Target audience
Migration factory

## Prerequisites
- Lab 01

## Estimated time
2-3 hours

## Tools required
- Inventory template, scripts

## Step-by-step lab
1. Use `samples/migration-data/sample-repo-inventory.csv`.
2. Populate repo size, branches, PRs, pipelines.
3. Tag high-risk repos.
4. Assign target GitHub org and repo name.

## Common mistakes
- Missing default branch or last commit date

## Evidence to submit
- Completed inventory file

## Review checklist
- Inventory is complete and consistent

## Definition of done
- Inventory supports wave planning
