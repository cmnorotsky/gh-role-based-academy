# Lab 04: Dependency Review

## Objective
Use dependency review in PRs.

## Target audience
Security, app teams

## Prerequisites
- Lab 01

## Estimated time
1-2 hours

## Tools required
- GitHub Actions

## Step-by-step lab
1. Add a dependency review workflow.
2. Create a PR that changes dependencies.
3. Review findings and document action.

## Common mistakes
- Ignoring critical vulnerabilities

## Evidence to submit
- PR link and review notes

## Review checklist
- Findings are triaged

## Definition of done
- Dependency review runs on PR
