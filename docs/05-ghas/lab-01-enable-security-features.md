# Lab 01: Enable Security Features

## Objective
Enable GHAS features for a repo.

## Target audience
Security, app teams

## Prerequisites
- GHAS module overview

## Estimated time
1-2 hours

## Tools required
- GitHub web UI

## Step-by-step lab
1. Navigate to repository settings.
2. Enable code scanning, secret scanning, and dependency review.
3. Document which features are available by license.

## Common mistakes
- Assuming features are available without licensing

## Evidence to submit
- Screenshot or note confirming enabled features

## Review checklist
- Features align with policy

## Definition of done
- Security features enabled or licensing gap documented
