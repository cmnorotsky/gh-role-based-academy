# GHAS Module Overview

## Objective
Enable and operate GitHub Advanced Security.

## Target audience
Security engineers, app teams

## Prerequisites
- Foundations, Actions basics

## Estimated time
8-10 hours

## Labs
1. Enable security features
2. Code scanning with CodeQL
3. Secret scanning and push protection
4. Dependency review
5. Dependabot
6. Security campaigns
7. Security metrics

## Deliverables
- GHAS rollout plan

## Review criteria
- Scanning enabled with clear triage workflow
