# Lab 02: Branching, PRs, and Reviews

## Objective
Practice PR workflow and reviews.

## Target audience
All roles

## Prerequisites
- Lab 01

## Estimated time
2 hours

## Tools required
- Git, VS Code, GitHub

## Step-by-step lab
1. Create a branch `learner/<your-name>/pr-review`.
2. Edit your evidence file with a new section.
3. Push changes and open a PR.
4. Request a mentor review.
5. Address feedback and merge.

## Common mistakes
- Missing reviewer request
- Merging without review

## Evidence to submit
- PR link
- Review comments addressed

## Review checklist
- PR description is clear
- Changes are small and focused

## Definition of done
- PR merged with approval
