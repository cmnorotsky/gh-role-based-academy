# Lab 04: Repo Settings and Protections

## Objective
Understand repo settings, branches, and protections.

## Target audience
All roles

## Prerequisites
- Lab 02

## Estimated time
1-2 hours

## Tools required
- GitHub web UI

## Step-by-step lab
1. Review existing branch protection rules.
2. Identify required status checks.
3. Document required review rules.
4. Propose a ruleset change in markdown.

## Common mistakes
- Confusing rulesets with branch protections

## Evidence to submit
- Proposed ruleset change in evidence file

## Review checklist
- Proposal aligns with governance model

## Definition of done
- Mentor reviews and approves proposal
