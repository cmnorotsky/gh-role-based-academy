# Lab 01: Requirements as Code

## Objective
Create requirements in markdown linked to issues.

## Target audience
BA/Product, architects

## Prerequisites
- Foundations

## Estimated time
2-3 hours

## Tools required
- GitHub Issues, Copilot Chat

## Step-by-step lab
1. Create an epic issue.
2. Create three user stories.
3. Add acceptance criteria and test notes.
4. Add requirements markdown under `requirements/`.
5. Link issues to the markdown file.

## Common mistakes
- Vague acceptance criteria

## Evidence to submit
- Epic, stories, and requirements file

## Review checklist
- Requirements are testable

## Definition of done
- Requirements approved
