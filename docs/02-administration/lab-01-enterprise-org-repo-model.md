# Lab 01: Enterprise, Org, and Repo Model

## Objective
Define the GitHub enterprise structure.

## Target audience
Platform, DevOps, GitHub admins

## Prerequisites
- Foundations module

## Estimated time
2 hours

## Tools required
- GitHub web UI

## Step-by-step lab
1. Review existing orgs and repos (or use TODO placeholders).
2. Draft target org structure by business unit or domain.
3. Define shared services org for templates and actions.
4. Propose repo naming standards.
5. Document org ownership and admin roles.

## Common mistakes
- Creating too many orgs without clear ownership

## Evidence to submit
- Org model diagram or markdown

## Review checklist
- Clear ownership and separation of concerns

## Definition of done
- Org model reviewed by mentor
