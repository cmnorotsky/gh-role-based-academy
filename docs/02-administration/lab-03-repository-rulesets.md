# Lab 03: Repository Rulesets

## Objective
Define repository rulesets for consistent governance.

## Target audience
Platform, DevOps, GitHub admins

## Prerequisites
- Lab 01

## Estimated time
1-2 hours

## Tools required
- GitHub web UI

## Step-by-step lab
1. Identify required branch protections.
2. Define required status checks.
3. Define review requirements and code owners.
4. Document exceptions and escalation.

## Common mistakes
- Rulesets that block emergency fixes without process

## Evidence to submit
- Ruleset plan in markdown

## Review checklist
- Aligns with governance model and security needs

## Definition of done
- Ruleset plan approved
