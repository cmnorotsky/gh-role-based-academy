# Lab 04: Audit Log and Governance

## Objective
Define audit log review and governance cadence.

## Target audience
Platform, security, GitHub admins

## Prerequisites
- Lab 01

## Estimated time
1-2 hours

## Tools required
- GitHub audit log (enterprise)

## Step-by-step lab
1. Identify audit events required for compliance.
2. Define review cadence and owners.
3. Draft escalation process.
4. Document audit retention requirements.

## Common mistakes
- No ownership for audit reviews

## Evidence to submit
- Audit log review plan

## Review checklist
- Clear owners and escalation path

## Definition of done
- Governance board approves the plan
