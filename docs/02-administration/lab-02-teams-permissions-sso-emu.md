# Lab 02: Teams, Permissions, SSO, EMU

## Objective
Define team structure and identity model.

## Target audience
Platform, DevOps, GitHub admins

## Prerequisites
- Lab 01

## Estimated time
2 hours

## Tools required
- GitHub web UI

## Step-by-step lab
1. Identify required teams and roles.
2. Map ADO groups to GitHub teams.
3. Decide EMU usage and identity mapping.
4. Document SSO/SAML/SCIM requirements.
5. Create a permission matrix draft.

## Common mistakes
- Assigning direct user permissions instead of teams

## Evidence to submit
- Team and permission matrix

## Review checklist
- Least privilege and EMU awareness

## Definition of done
- Mentor approves identity and team model
