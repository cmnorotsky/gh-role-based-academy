# Lab 03: Environments and Approvals

## Objective
Use environments and approvals for production.

## Target audience
DevOps, platform

## Prerequisites
- Lab 01

## Estimated time
2 hours

## Tools required
- GitHub Actions environments

## Step-by-step lab
1. Define a `production` environment in GitHub.
2. Add required reviewers.
3. Create a workflow that deploys to `production`.
4. Trigger the workflow and request approval.

## Common mistakes
- Using environment without reviewers

## Evidence to submit
- Screenshot or note of approval gate

## Review checklist
- Approval gate enforced

## Definition of done
- Deployment requires approval
