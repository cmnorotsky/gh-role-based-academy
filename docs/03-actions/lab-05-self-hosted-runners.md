# Lab 05: Self-Hosted Runners

## Objective
Plan self-hosted runner usage safely.

## Target audience
DevOps, platform

## Prerequisites
- Lab 01

## Estimated time
1-2 hours

## Tools required
- GitHub Actions

## Step-by-step lab
1. Identify workloads needing self-hosted runners.
2. Define runner groups and labels.
3. Document security controls and patching.
4. Propose autoscaling approach.

## Common mistakes
- Running untrusted workflows on shared runners

## Evidence to submit
- Runner strategy document

## Review checklist
- Security controls and isolation documented

## Definition of done
- Runner strategy approved by platform lead
