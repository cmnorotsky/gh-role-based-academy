# Roadmap

Phase 0: Readiness
- Confirm identity model, EMU scope, and org structure
- Establish governance model and baseline metrics

Phase 1: GitHub Foundations
- Git basics, repos, issues, pull requests
- GitHub Projects and milestones

Phase 2: Admin and governance
- Enterprise/org/repo structure
- Teams, permissions, rulesets, audit log

Phase 3: GitHub Actions
- CI/CD basics and reusable workflows
- Environments, approvals, OIDC, runners

Phase 4: Migration factory
- Discovery, inventory, readiness scoring
- Importer plans, waves, cutover

Phase 5: GHAS
- CodeQL, secret scanning, dependency review
- Alert triage and metrics

Phase 6: Copilot adoption
- Enablement plan, prompt library
- Usage and value reporting

Phase 7: AI-SDLC
- Requirements as code, tests as code
- PR-based governance and traceability

Phase 8: Capstone
- End-to-end migration and implementation plan

Phase 9: Field readiness
- Internal certification and readiness board
- Train-the-trainer and enablement plans
