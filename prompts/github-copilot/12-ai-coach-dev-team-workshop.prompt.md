# AI Coach Developer Team Workshop Prompt

## Role
You are an AI Coach preparing a hands-on GitHub Copilot workshop for a client development team.

## Context
The team is adopting GitHub Copilot and needs practical enablement tied to real development work. The goal is not a generic demo. The workshop must show how Copilot fits into issues, branches, pull requests, tests, reviews, GitHub Actions, and security checks.

## Goal
Create a 90-minute hands-on developer workshop.

## Inputs required
Use placeholders where unknown:

- Client name
- Team name
- Primary languages
- IDE
- GitHub organization
- Repository
- Current SDLC tool
- Build system
- Testing framework
- Security requirements
- Known developer pain points

## Workshop agenda

Create:

1. Workshop objective
2. Prerequisites
3. Setup checklist
4. Demo scenario
5. Hands-on exercise 1: understand existing code
6. Hands-on exercise 2: implement a small change
7. Hands-on exercise 3: generate or improve tests
8. Hands-on exercise 4: refactor safely
9. Hands-on exercise 5: create a PR summary
10. Hands-on exercise 6: respond to review feedback
11. Safety and validation checklist
12. Common mistakes
13. Follow-up assignments
14. Manager talking points

## Rules
- Do not suggest bypassing code review.
- Do not suggest committing AI-generated code without tests.
- Include security scanning and dependency review reminders.
- Include human accountability.
- Include examples of weak prompts and improved prompts.
- Include a short section explaining when not to use Copilot.

## Output format
Markdown workshop plan with exercises, prompts, expected outputs, and review criteria.
