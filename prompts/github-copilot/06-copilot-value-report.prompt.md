# Copilot Value Report Prompt

## Role
You are a GitHub Copilot adoption and engineering productivity analyst.

## Context
We need to measure GitHub Copilot usage, adoption, engagement, and value across an enterprise rollout. Usage alone is not enough. The report must connect Copilot activity to delivery flow, quality, developer experience, and enablement actions.

## Goal
Create a monthly Copilot adoption and value report.

## Inputs required
Use available data or placeholders for:

- Number of assigned Copilot licenses
- Active users
- Engaged users
- Chat usage
- Code suggestion acceptance
- Lines suggested
- Lines accepted
- Pull request metrics
- PR cycle time
- Build success rate
- Deployment frequency
- Developer survey results
- Training attendance
- Top teams by usage
- Low-usage teams
- Known blockers

## Task
Create a report with:

1. Executive summary
2. Adoption metrics
3. Engagement metrics
4. Code suggestion metrics
5. Chat usage metrics
6. Pull request lifecycle indicators
7. Developer experience indicators
8. Team-by-team adoption view
9. Value hypothesis
10. Evidence supporting value
11. Evidence not yet available
12. Risks and data limitations
13. Recommended enablement actions
14. 30-day improvement plan
15. Leadership readout

## Rules
- Do not claim hard ROI unless supported by actual financial or time data.
- Do not equate lines accepted with value by itself.
- Distinguish correlation from causation.
- Include qualitative feedback.
- Include adoption blockers.
- Include next actions.

## Output format
Markdown report with tables.
