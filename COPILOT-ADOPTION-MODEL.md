# Copilot Adoption Model

## Purpose
Provide a safe, measurable enterprise rollout plan for GitHub Copilot.

## Pilot selection
- Select 2-4 teams with active delivery work
- Include mixed roles and domains

## Persona mapping
- Map dev, QA, BA, DevOps, and security personas

## License assignment
- Assign licenses to pilot users
- Confirm IDE access and extensions

## Prompt enablement
- Provide role-based prompt library
- Include approved prompt patterns and anti-patterns

## IDE setup
- VS Code configuration and extensions
- Verify Copilot Chat works in IDE

## Responsible AI guidance
- Human review required
- Do not paste secrets or restricted data

## Data and privacy guidance
- Follow enterprise policy and data classification
- Use TODO placeholders for org policy links

## Usage dashboards
- Track active users, engagement, acceptance rate
- Use GitHub usage exports or enterprise dashboards

## Value reporting
- Combine usage with delivery flow and quality metrics

## Adoption risks
- Lack of training, unclear policies, poor IDE setup
- Over-reliance on AI without review

## Mitigation actions
- Office hours, prompt coaching, role-based labs

## 30/60/90-day success criteria
- 30 days: Pilot teams active, baseline metrics captured
- 60 days: Prompt library and role-based enablement complete
- 90 days: Adoption trend positive, value report delivered
